# Wyckoff 系统迭代策略

> 基于策略衰减理论（见 [STRATEGY_DECAY.md](./STRATEGY_DECAY.md)），规划系统级对抗衰减能力。
> 当前阶段：**规划中，暂不实现**。需额外存储机制支撑，待存储压力缓解后推进。

## 方向一：信号衰减监控

**目标**：按信号类型追踪推荐质量，识别正在失效的信号。

**思路**：
- 按信号类型（SOS/Spring/LPS/EVR/Compression）分别统计近 N 期胜率和平均收益
- 信号连续 M 期胜率低于阈值 → 自动降权（非下线）
- 区分"信号衰减"和"regime 不匹配"——CRASH 下 Trend 信号天然弱势不等于信号坏了
- 定期输出信号健康度报告

**前置依赖**：
- 需要持久化每次推荐的信号类型、推荐价、后续涨跌（现有 reprice job 部分覆盖）
- 需要按信号类型聚合的历史存储（当前 Supabase 存储压力大，暂缓）

## 方向二：多策略动态分配

**目标**：AI 候选配额从静态切换为数据驱动的动态权重。

**思路**：
- 现状：`resolve_ai_candidate_policy` 按 regime 硬切 trend/accum 配额（如 RISK_OFF: 2/3）
- 目标：配额 = f(regime, 各信号近期表现, 市场广度变化趋势)
- regime 切换时渐进调权，避免硬切造成的信号真空
- 信号衰减时自动减少该方向输出量

**前置依赖**：
- 方向一的信号健康度数据
- 回测验证动态分配 vs 静态分配的收益差异

## 方向三：信号生命周期管理

**目标**：建立新信号孵化 → 正式上线 → 衰减降级的完整通道。

**思路**：
- 新信号进"实验池"（类似 L2 旁路观察），只跟踪不推荐
- 达到统计显著（如 30+ 样本，胜率 > 基线 + 1σ）后晋升正式信号
- 老信号衰减到阈值后降级为观察态，继续收集数据但不占 AI 配额
- 形成"研究 → 实验 → 正式 → 观察 → 退役"的闭环

**前置依赖**：
- 方向一 + 方向二落地后
- 信号实验框架（独立于主漏斗的轻量评估管线）

## 优先级

```
方向一（监控） → 方向二（动态分配） → 方向三（生命周期）
     ↑
  等存储方案就绪后启动
```

## Wyckoff 系统的天然优势

对照衰减速度表，Wyckoff 体系属于"衰减慢"一侧：
- ✅ 中长期持仓（日线级别）
- ✅ 多因子复合系统（量价结构 + 阶段 + 板块 + regime）
- ✅ 自研事件库（Wyckoff 形态识别非公开因子）
- ⚠️ 执行可见度中等（每日定时推送，有一定规律性）
- ⚠️ 标的池覆盖 A 股全市场，非冷门标的

最大风险不是单一信号衰减，而是整个 Wyckoff 形态识别体系被更多量化团队复现。对策是持续深化对市场结构的理解，保持信号定义的独特性。
